前言

学习过很多东西，但是很久不用就容易忘，只是记在笔记上太没意思了，在网上看到过很多好看的博客，很符合我这又有精神洁癖又爱折腾的性格，以前尝试做过一次博客，但是没有坚持下去，现在打算重启，这个系列用于记录Hexo和GitHub搭建博客的过程，及用于记录，也是分享。

什么是Hexo？

Hexo是一款基于Node.js的静态博客框架，依赖少，容易安装使用，可以方便生成静态网页托管在GitHub和Heroku上

搭建步骤

  • GitHub创建个人仓库
  • 按照Git
  • 安装Node.js
  • 安装Hexo
  • 个性化
  • 发布文章
  • 推送网站

GitHub创建个人仓库

登录GitHub，点击GitHub中New repository创建新仓库，仓库名为：用户名.github.io这个用户名使用你的GitHub账号名代替，比如我的仓库名为：

yancyshawn.github.io

安装Git

什么是Git？简单说Git是开源的分布式版本控制系统。我们网站在本地搭建好后，需要使用Git同步到GitHub上。从Git官网下载：Git-Download for Windows 官网下载速度较慢，可以使用国内阿里云镜像-git for windows
安装成功后，需要将Git和GitHub账号绑定，右键鼠标，点击Git Bash Here。

或者在菜单里搜索Git Bush，然后设置user.name和user.email配置信息

git config --global user.name "GitHub用户名"
git config --global user.email "GitHub注册邮箱"

生成ssh秘钥文件

ssh-keygen -t rsa -C "GitHub注册邮箱"

然后直接三个回车即可，默认不需要设置密码
然后找到生成的**.ssh文件夹中id_rsa.pub**秘钥


打开GitHub-Setting-keys 页面，新建new SSH Key


Title为标题，将刚刚复制的id_rsa.pub粘贴进去，点击Add SSH Key.
在Git Bash检测GitHub公钥设置是否成功

ssh git@github.com


出现如上情况，说明成功。
此处设置秘钥的原因是通过非对称加密的公钥和私钥来完成加密，公钥放在GitHub上，私钥设置在电脑上。GitHub要求每次推送代码都是合法用户，所以每次推送都需要输入账号密码验证推送用户是否合法，为了省去每次输入密码的步骤，才用ssh，当你推送的时候，git就会匹配你的私钥跟GitHub上的公钥是否成功配对，若匹配，则是合法用户，允许推送。

安装Node.js

Hexo基于Node.js，Node.js下载地址：Node.js-Download ，注意安装Node.js会包含环境变量和npm的安装，检测Node.js是否安装成功，在CMD里输入:

node -v


检测npm是否安装成功，在CMD里输入:

npm -v

到此为止，安装Hexo的环境已基本完成
